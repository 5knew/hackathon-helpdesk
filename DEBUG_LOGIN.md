# Отладка входа

## Добавлено логирование

В `backend/routers/auth.py` добавлено логирование для отладки:
- Входящий email и длина пароля
- Нормализованный email
- Найден ли пользователь
- Совпадает ли пароль

## Проверка

1. Перезапустите бэкенд (если не используете --reload)
2. Попробуйте войти через фронтенд
3. Проверьте логи бэкенда в терминале - там будет видно, что происходит

## Возможные проблемы

1. **Email не нормализуется правильно** - проверьте логи
2. **Пароль не совпадает** - проверьте, какой пароль сохранен в базе
3. **Проблема с кодировкой** - символ "?" может кодироваться по-разному

## Быстрое решение

Если логи показывают, что пользователь не найден, проверьте реальный email в базе:
```python
python backend/check_user.py
```

Если пароль не совпадает, обновите его:
```python
python backend/update_password.py
```
